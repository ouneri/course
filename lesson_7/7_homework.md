# Домашнее задание 7: TypeScript

## Цель

Перевести ваш проект портфолио на TypeScript, добавив профессиональную типизацию и улучшив существующий функционал. Это не просто учебное задание - это создание полноценного TypeScript проекта для вашего портфолио, который покажет реальные навыки современной фронтенд-разработки.

## Обзор задания

На основе вашего текущего проекта портфолио (с интерактивностью на JavaScript) вы создадите TypeScript-версию с серьезными улучшениями. Проект станет надежнее, масштабируемее и профессиональнее - именно то, что ожидают от современного фронтенд-разработчика.

## Задания

### Небольшой рефакторинг файловой структуры репозитоиия

Предлагаю перенести текущий веб-проект, в котором производится работа, в директорию portfolio_website в корне репозитория, тогда репозиторий будет выглядеть примерно следующим образом:

- lesson_1/
- lesson_2/
- ...
- lesson_7/
- portfolio_website

### Миграция JS -> TS

Требуется разобраться с настройкой Vite для работы с TypeScript. Соответственно, следует доработать текущий функционал, добавив в него типизацию.

### Mock как аналог работы с сервером

Сделайте данные, которые отображаются на странице не статическими (не вписанными в разметку), а динамическими - будто они приходят с сервера. Для этого заведите модель Skill (или ISkill - как больше нравится), заведите ISkillService - абстракцию, интерфейс, которые содержит в себе метод getSkills(): Promise<readonly ISkill[]> (или, например, Promise<IPagination<ISkill>> - если хочется сделать ещё более продвинутый способ), и реализуйте MockSkillService - который будет ПОКА ЧТО возвращать статически вписанные данные. Очевидно, такого рода доработка требует видоизменения файловой структуры репозитория, меняйте его на своё усмотрение, лишь бы ни в одном main.ts файле - я бы делал src/models, src/services и т.д. Подумайте, какие ещё данные возможно сделать `динамическими`.

### Система тем

Сделайте систему работы с темами на сайте. Для этого помогут CSS переменные, localStorage (для сохранения тем между сессиями, для сохранения текущей выбранной темы), пусть по умолчанию будет 2 темы - тёмная и светлая, реализуйте создание, сохранение собственной. Так же, по аналогии с получением навыков - IThemeService (якобы работа с сервером - по факту local-first). Интерфейсы лучше проектировать с упором на то, что работа идёт с сервером, поэтому пусть методы возвращают Promise. UI на усмотрение разработчика. Здесь немало пространства для применения навыков из ООП - так, например, можно описать цветовое значение строкой, можно описать отдельным классом, например, - HexColor - со своими правилами для валидации и т.д.
